## 核心数据模型

4 层关联：**属性(18) → 技能(469+) → 蛋组(15) → 精灵(466+)**

| 数据 | UID 格式 | 存储 |
|------|----------|------|
| 属性 | `elem_{id}` | data/elements/ |
| 技能 | `skill_{N}` | data/skills/ |
| 蛋组 | 按名称 | data/eggs/ |
| 精灵 | `pet_{id}` 或 `pet_{id}_{N}` | data/pets/ |

精灵字段包含：element(属性引用)、egg_groups(蛋组名称列表)、detail(含技能/克制/立绘)

---

## 爬虫 (crawler/)

- 入口：`crawler/run.py --full` / `--update`
- 并发：详情页 5 线程(0.5s)，图片 10 线程(0.1s)
- 限流：567/429 自动重试(30s×次数，最多3次)
- 执行顺序：属性 → 技能 → 蛋组 → 精灵列表 → 精灵详情
- 增量模式：前置数据每次都重新拉取；精灵详情按 version 判断，但映射关系(egg_groups/element/skill_ref)始终刷新
- 完成后自动：打印完整性汇总 + 同步 SQLite 数据库

## 后台 (app/server/)

- Express + better-sqlite3
- Service 层独立（可复用到 Electron IPC）
- RESTful API：`/api/elements`、`/api/skills`、`/api/eggs`、`/api/pets`
- 静态资源：`/public/...` 指向 `data/public/`
- 前端托管：构建产物在 `server/public/`，SPA fallback

### 关键命令
```bash
npm run setup        # init-db + import-data
npm run dev          # 启动 Express :3000
```

---

## 前端 (app/client/)

- Vue 3 + Vue Router + Vite
- TailwindCSS + Sass
- 构建输出到 `app/server/public/`（生产模式单服务部署）
- 开发模式 :5173 代理 API 到 :3000
- 字体：MIANFEIZITI（洛克专用免费字体，从腾讯 CDN 加载）

### 页面
| 路由 | 页面 |
|------|------|
| `/` | 首页概览 |
| `/pets` | 精灵图鉴（搜索/筛选/排序/分页） |
| `/pets/:uid` | 精灵详情（技能/克制/蛋组） |
| `/skills` | 技能大全 |
| `/elements` | 属性克制关系 |

## 图片路径规范

所有 JSON 中的图片路径为本地 `/public/...` 格式：
- 属性图标：`/public/elements/icons/elem_{id}.png`
- 技能图标：`/public/skills/icons/skill_{N}.png`
- 精灵缩略图：`/public/pets/thumbnails/{uid}.png`
- 精灵立绘：`/public/pets/default/{uid}_default.png`
- 异色立绘：`/public/pets/shiny/{uid}_shiny.png`
- 果实图片：`/public/pets/fruit/{uid}_fruit.png`
- 精灵蛋：`/public/pets/egg/{uid}_egg.png`

## 重要约定

1. `data/` 目录不纳入 git（含大量图片）
2. 爬虫完成后自动同步 SQLite（需 server 已 npm install）
3. 增量模式下即使详情不重新爬取，也会刷新所有映射关系
4. 前端 build 产物输出到 `app/server/public/`，server 自动托管
5. Service 层不依赖 Express，后续可无缝迁移到 Electron IPC
