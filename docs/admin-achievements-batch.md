# 图鉴课题批量管理

## 一、功能定位

在管理端新增一个独立的**「图鉴课题」**列表页面（`/admin/achievements`），以精灵为单位、按精灵编号（`pet_id`）顺序展示所有精灵的图鉴课题配置，支持分页浏览和快速编辑，解决当前只能逐个进入精灵详情页配置课题的效率问题。

## 四、核心功能

| 功能 | 说明 |
|------|------|
| **列表展示** | 按 `pet_id` 排序展示所有精灵，每条显示缩略图、名称、UID、属性图标、标签（最终形态/首领形态/拥有首领形态）、课题摘要 |
| **分页** | 每页 **20 只**精灵（课题信息较多，20条平衡信息密度和加载速度） |
| **筛选** | 支持名称/UID搜索、属性筛选、课题状态筛选（全部/仅默认/有自定义/无课题） |
| **课题摘要** | 显示默认课题数量+标题列表、自定义课题数量+类型标识（🎯技能课题/📝文本课题） |
| **快速编辑** | 点击「编辑课题」展开内联编辑面板（不跳转页面），可添加/删除自定义课题、切换默认课题显隐 |
| **跳转详情** | 点击「跳转详情」跳转到 `/admin/pets/:uid` 精灵编辑页 |
| **隐藏状态** | 默认课题如有隐藏的，显示删除线样式+灰色文字 |
| **首领形态** | `is_boss_form=1` 的精灵直接显示提示文字，不展示编辑按钮 |

## 五、后端API

新增一个批量获取接口：

**`GET /api/admin/achievements-list`**

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` | number | 页码，默认1 |
| `limit` | number | 每页数量，默认20 |
| `search` | string | 搜索关键词（名称/UID） |
| `element_id` | number | 属性筛选 |
| `achievement_filter` | string | 课题状态：`all`/`default_only`/`has_custom`/`no_achievements` |

**返回格式**：
```json
{
  "pets": [
    {
      "uid": "pet_001",
      "name": "喵喵",
      "pet_id": "1",
      "thumb_url": "/pets/thumbs/pet_001_default.webp",
      "element_icon": "/elements/icons/elem_2.png",
      "sub_element_icon": null,
      "is_final_form": 0,
      "is_boss_form": 0,
      "has_boss_form": 0,
      "default_count": 3,
      "custom_count": 0,
      "hidden_count": 0,
      "achievements": [...]
    }
  ],
  "total": 466
}
```
